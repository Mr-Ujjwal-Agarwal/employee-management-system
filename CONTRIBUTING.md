# Contributing

Thank you for your interest in contributing to this project.

## Branch Naming

feature/<feature-name>

bugfix/<bug-name>

docs/<documentation>

hotfix/<issue>

---

## Commit Convention

Examples

feat(auth): implement JWT authentication

fix(api): resolve employee update bug

docs(readme): improve architecture section

refactor(terraform): simplify VPC module

---

## Pull Requests

Every pull request should:

- Follow commit conventions
- Pass all tests
- Include documentation updates
- Be reviewed before merge

---

## Code Style

### Python

- Follow PEP8
- Use meaningful variable names
- Add docstrings

### React

- Functional Components
- Hooks
- ESLint compliant

### Shell Scripts

- Use bash
- Add comments
- Fail fast (`set -e`)

---

## Infrastructure

Infrastructure changes should:

- Update documentation
- Be tested
- Avoid breaking production resources

---

Happy Coding 🚀
